# Fix 403 Permission Denied Error

## The Problem

You're getting `403 Permission denied` which means:
- The token doesn't have the right permissions, OR
- The repository doesn't exist, OR  
- The token is invalid

## Step 1: Verify Repository Exists

1. Go to: **https://github.com/arukurmi/Log-zilla**
2. Make sure the repository exists and you can see it
3. If it doesn't exist, create it first at: https://github.com/new

## Step 2: Check Token Permissions

1. Go to: **https://github.com/settings/tokens**
2. Find your token (the one starting with `github_pat_11AQAUEKY...`)
3. Click on it to see its permissions
4. **Make sure it has:**
   - ✅ `repo` (Full control of private repositories) - **REQUIRED**
   - ✅ `workflow` (optional, for GitHub Actions)

## Step 3: Create a NEW Token with Correct Permissions

If the token doesn't have `repo` permission:

1. Go to: **https://github.com/settings/tokens**
2. Click **"Generate new token"** → **"Generate new token (classic)"**
3. Fill in:
   - **Note:** "Log-zilla Push Access"
   - **Expiration:** Your choice (90 days, 1 year, or no expiration)
   - **Scopes:** Check **`repo`** (this is the most important one!)
4. Click **"Generate token"**
5. **Copy the new token** (you won't see it again!)

## Step 4: Revoke the Old Token

1. Go back to: **https://github.com/settings/tokens**
2. Find the old token (`github_pat_11AQAUEKY...`)
3. Click **"Revoke"** to delete it (since it was shared publicly)

## Step 5: Push with New Token

```bash
cd /Users/aryanshkurmi/code/Log-zilla

# Clear old credentials
git credential-osxkeychain erase <<EOF
host=github.com
protocol=https
EOF

# Push (will prompt for username and new token)
git push -u origin main
```

When prompted:
- **Username:** `arukurmi`
- **Password:** Paste your NEW token

## Alternative: Test Token First

Test if your token works:

```bash
curl -H "Authorization: token YOUR_NEW_TOKEN" https://api.github.com/user
```

This should return your GitHub user info if the token is valid.
